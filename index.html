Implementation plan
01
Install Paperclip on thinkcentre
Day 1 · ~30 min

Paperclip is an npm package, just like OpenClaw. Install it on the same thinkcentre server — it'll share the Tailscale network and can reach your OpenClaw gateway directly. No new infrastructure needed.

  • SSH into thinkcentre as jbota: ssh jbota
  • Run the onboard wizard: npx paperclipai onboard --yes
  • Set up the database (SQLite is fine to start) and auth when prompted
  • Open the Paperclip UI in the browser (Tailscale-accessible URL it gives you)
  • Verify it's running: check the dashboard loads
# on thinkcentre (ssh jbota)
npx paperclipai onboard --yes
# follow the interactive setup for DB + auth
# it will give you a local URL to open
02
Create your company and set the goal
Day 1 · ~15 min

In Paperclip, every company starts with a mission statement. This is what TheOne will read every time it wakes up on a heartbeat. Keep it specific and measurable.

  • In the Paperclip UI, click "New company"
  • Name: Outreach Co. (or whatever fits)
  • Goal: "Generate qualified B2B leads via automated LinkedIn outreach and personalised reply generation — targeting 50 qualified contacts per week"
  • Set a monthly budget cap (start with €100 to be safe)
  • You are the board operator — no other users needed
03
Register TheOne as CEO agent
Day 1 · ~20 min

Add TheOne as the first employee. Use the openclaw_gateway adapter and point it at your existing gateway URL. Paperclip will send it a heartbeat payload — OpenClaw receives it and wakes TheOne up exactly as it does now via Telegram.

  • In Paperclip UI → Employees → Hire agent
  • Name: TheOne · Title: CEO · No manager (reports to board)
  • Adapter type: openclaw_gateway
  • Gateway URL: https://thinkcentre.tail13ae6a.ts.net
  • Agent ID: main
  • Set monthly budget: €40 (Gemini Flash is cheap)
  • Trigger a manual heartbeat and confirm TheOne wakes up in Telegram
Board approval gate: After TheOne's first heartbeat it will produce a strategic plan and propose hiring Scout, Spark, and the pipeline agent as subordinates. You'll see an approval request in the Paperclip UI. Review and approve each one.
04
Register Scout and Spark as direct reports
Day 2 · ~30 min

Same process as TheOne, but with different agent IDs and roles. Scout and Spark already exist in OpenClaw — you're just giving Paperclip the keys to wake them up.

  • Scout: adapter openclaw_gateway, agent ID research, reports to TheOne
  • Spark: adapter openclaw_gateway, agent ID social, reports to TheOne
  • Replicate Scout's existing Competitive Radar cron as a Paperclip heartbeat: Monday 8am schedule
  • Replicate TheOne's Morning Briefing cron: weekdays 7am schedule
  • Test both heartbeats manually — confirm Telegram messages arrive as before
  • Once confirmed, disable the old OpenClaw cron jobs to avoid duplicate runs
Don't delete crons yet — disable them first (active: false in jobs.json) and run for 3 days in parallel. Only remove them once you're confident Paperclip heartbeats are firing reliably.
05
Register the LinkedIn pipeline as a process agent
Day 2–3 · ~45 min

This is the most impactful step. Your pipeline currently runs on its own schedule in PM2. Paperclip takes over as the scheduler — and starts creating issues for each processed commenter, which Spark picks up automatically.

  • In Paperclip UI → Employees → Hire agent
  • Name: LinkedIn Pipeline · Title: Data Engineer · Reports to TheOne
  • Adapter type: process
  • Command: ssh linkedin 'node /home/linkedin/linkedin-pipeline/pipeline.js'
  • Heartbeat: daily 8am (replaces node-cron in index.js)
  • Add a small modification to pipeline.js: after each comment is processed, POST to the Paperclip API to create an issue assigned to Spark
  • In PM2: stop the scheduler (pm2 stop linkedin-pipeline) — keep the process available but let Paperclip call it
  • Test with a single profile, verify issue appears in Paperclip and Spark wakes up
# small addition to pipeline.js after generateReply()
await fetch(`${process.env.PAPERCLIP_API_URL}/issues`, {
  method: 'POST',
  headers: { Authorization: `Bearer ${process.env.PAPERCLIP_API_KEY}` },
  body: JSON.stringify({
    title: `Reply to ${commenterName} on LinkedIn`,
    description: suggestedReply,
    assigneeId: 'spark-agent-id',
    status: 'in_review'
  })
});
06
Set up the in_review gate for Spark
Day 3 · ~20 min

Instead of writing directly to Pipedrive, Spark's output waits in Paperclip as an in_review issue. You approve in the UI → it moves to done and a webhook writes the Pipedrive activity. This gives you quality control and a full audit trail of every reply.

  • In Paperclip, configure Spark's issues to require board approval at in_review
  • Create a Paperclip webhook: on issue status → done, POST to a small Express endpoint on the Hostinger VPS
  • That endpoint calls your existing pipedrive.js createActivity function
  • Test the full loop: pipeline → Paperclip issue → Spark reply → you approve → Pipedrive activity
  • Update Spark's SOUL.md to reference the Paperclip issue context format
Optional upgrade: After a week of approvals, if Spark's quality is consistently good, remove the board gate and let issues move straight from in_review → done automatically. You've calibrated Spark's SOUL.md by then.
07
Run for 2 weeks, then decide on the startup angle
Week 2–3 · ongoing

With Paperclip running, you now have real data: which agents cost the most per task, what the pipeline outputs per day, how often you approve vs reject Spark's replies. This data is your pitch deck. Two angles worth evaluating once you have 2 weeks of data:

  • Productise your pipeline: Your LinkedIn enrichment → reply → CRM flow is a sellable SaaS. Paperclip's Clipmart lets you export the whole company structure as a template others can import.
  • Offer it as a service: Run the pipeline for other founders/sales teams on their LinkedIn profiles. Paperclip's multi-company support means one install can serve 10+ clients with full data isolation.
  • Export a Clipmart template of your company structure once it's stable
  • Document cost per lead (Apify + OpenRouter + Gemini per qualified contact)
  • Identify the one workflow that saves the most time → that's your wedge

Paperclip is open-source and fully self-hosted — it adds €0 to your monthly cost. The only new overhead is the thinkcentre running one more Node.js process, which it can handle easily.

At $51/mo, if your pipeline qualifies even 5 leads per month that convert to any kind of deal, the economics already work. The Paperclip layer makes this auditable and sellable.
